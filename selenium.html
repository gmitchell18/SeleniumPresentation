Automated Testing with Selenium

What is Selenium

  • Selenium is a portable software testing framework for web applications

    • Provides a record/playback tool for authoring tests without learning a test scripting languages
    • Provides a test domain-specific language to write tests in a number of popular programming languages

  • Selenium Consists of four major components

    • Selenium IDE
    • Selenium Web Driver API
    • Selenium Remote Control
    • Selenium Grid

Components

  • Selenium IDE

    Integrated Development environment for Selenium Scripts

    Firefox-only extension that allows for record and playbacks of tests

  • Selenium Web Driver

    Drives browser natively

  • Selenium Remote Control

    Client/Server system that allows you to control browser through code

  • Selenium Grid

    Runs test on many servers at the same time

Benefits

  • Runs on almost every operating system
  • Open source testing tool
  • Supports multiple languages such as Java, Python, Perl, Ruby, PHP, .NET (C#)
  • Language used for building the test scripts are not coupled to the language that the application is using
  • Supports a range of browsers
  • Provides support for integration with open source frameworks
  • Allows for the execution of simultaneous tests leveraging various browsers on various machines

Drivers

Firefox

  • Included in selenium-standalone.jar

  • Comes in the form of an XPI (firefox extension)

  • Faster than IE Driver

Chrome

  • Standalone server which implements WebDriver's wire protocol

  • Download Chrome Driver

  • Requires Chrome is installed in default location

  • Faster than IE Driver

Internet Explorer

  • Standalone server that implements WebDriver's wire protocol

  • Download IEDriverServer

  • Works with IE 6 - 10

  • Only runs on Windows

Other Drivers

Selenium also supports

  • HTML

    Fastest implementation of Selenium WebDriver

    Major Disadvantage: does not test how the major web browsers will handle this application

  • Safari

  • Opera

// Give the Test class a descriptive name, so when it is bundled as part of a test suite (Java package), it is easy to determine what the test verifies
public class TestGoogleSuggestFirefox {
// Implemented in the main method. It could also be implemented as a JUnit class. 
    public static void main(String[] args) throws Exception {

        // The Firefox driver supports javascript 
        WebDriver driver = new FirefoxDriver();
        
        // Go to the Google Suggest home page
        driver.get("http://www.google.com/webhp?complete=1&hl=en");
        
        // Enter the query string "Cheese"
        WebElement query = driver.findElement(By.name("q"));
        query.sendKeys("Cheese");

        // Sleep until the div we want is visible or 5 seconds is over
        long end = System.currentTimeMillis() + 5000;
        while (System.currentTimeMillis() < end) {
            WebElement resultsDiv = driver.findElement(By.className("gssb_e"));

            // Suggestions should be displayed in a drop down
            if (resultsDiv.isDisplayed()) {
              break;
            }
        }

Running Selenium Test

  • Command Line 
    java TestGoogleSuggestFirefox
  • Integrated Development Environment
    • Scripts can be run through an IDE, such as Eclipse
    • Eclipse includes support for JUnit Test Framework
    • JUnit makes it easy to run tests and view the results, elpased time, and re-run failed tests
  • Build Tool 
    mvn -Dtest=TestGoogleSuggestFirefox test
    mvn -Dtest=Test*Firefox#test test
